Caso de Estudio: Auditor Inteligente de Medios

Pipeline Multimodal de Ingesta, Transcripción Inteligente y Auditoría Semántica Estructurada

⚠️ Estado del Despliegue en Vivo

El código del proyecto funciona al 100% y ha sido probado con éxito de manera local. Sin embargo, su ejecución automatizada interactiva a través de esta página web se encuentra limitada en este momento debido a los costes e infraestructuras de pago que los proveedores de nubes (como OpenAI y Google Cloud) exigen para procesar peticiones y análisis multimedia de uso público continuo.

1. ¿Cómo funciona el proyecto? (Explicación Sencilla)

Este sistema actúa como un "auditor robótico de noticias". Su tarea es recibir cualquier enlace de video, escuchar su contenido de forma interna, transformarlo en texto editable y procesar ese texto de forma matemática para descubrir la inclinación, sesgo o sentimiento del locutor de manera inmediata.

📥

1. Extracción

La herramienta usa yt-dlp para descargar el audio puro de videos en internet de forma automática.

🎙️

2. Transcripción

El modelo de Inteligencia Artificial **OpenAI Whisper** escucha el audio localmente y lo convierte en texto escrito.

🧠

3. Auditoría

Un algoritmo heurístico en Python lee la densidad de palabras clave y determina si el video tiene sesgo comercial o político.

2. Retos Técnicos Superados

Durante el desarrollo surgieron problemas complejos de integración que simulan el entorno real de un Ingeniero de Datos:

El Reto de los Archivos de Audio

Python por sí solo no sabe cómo procesar o transformar flujos multimedia complejos descargados directamente de internet.

La Solución

Se integró el software de procesamiento multimedia FFmpeg en las variables del sistema del entorno, lo que permitió automatizar la conversión binaria hacia canales MP3 estables.

Restricción de Cuotas Externas

Las llamadas iniciales a los servidores de APIs externas de Inteligencia Artificial arrojaban errores de saturación (Error 429) e interrupciones de acceso.

La Solución

Se rediseñó la arquitectura hacia un modelo Offline-First, implementando un motor analítico heurístico local que calcula las métricas semánticas directo en el hardware de la máquina de forma independiente y gratuita.

3. Demostración y Repositorio

El backend estructurado genera un reporte impecable en formato Markdown llamado informe_auditoria.md cada vez que el script finaliza.

Puedes revisar todo el código fuente modular, las variables lógicas de control y las instrucciones de instalación en mi repositorio público de GitHub.
